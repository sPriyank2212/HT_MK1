ARCHITECTURE (key fact)
--------------------------------------------------------------------------------
Single-MCU design. The STM32G474 on the Control Card is the ONLY processor in
the instrument. The Matrix Card and HV Cards have NO MCU - they are just I2C
GPIO expanders (MCP23017), analogue muxes (CD4067), reed relays, DACs and ADCs
driven by this one firmware over I2C / SPI / GPIO. All test logic lives here.

Code layout (new layers compile automatically - "Core" is a recursive CubeIDE
source path; include as "drivers/xxx.h" / "cards/xxx.h"):
    Core/Inc/drivers, Core/Src/drivers   - device drivers
    Core/Inc/cards,   Core/Src/cards     - per-card abstraction
    (planned) Core/.../test              - continuity / kelvin / insulation

SPI ALLOCATION (verified from .ioc pins + Control_Card isolator wiring)
--------------------------------------------------------------------------------
    SPI2  PB13/14/15        -> DAC8775 (Kelvin Idac), via isolators ("SPI1_ISO")
    SPI3  PB3/4/5 +CS        -> HV cards: DAC8830 + AD7476, via isol ("SPI2_ISO")
    SPI1  PA5/6/7            -> AD7476 on Control Card (the free SPI)  [ASSIGNED]
    Note: the SPI1_ISO/SPI2_ISO net names are crossed vs the MCU instance; by
    PINS, SPI2->DAC8775, SPI3->HV, SPI1 free. Per-device SPI settings below.

    SPI2 (DAC8775)       : 8-bit (24-bit frame sent as 3 bytes), <= ~50 MHz ok.
    SPI3 (DAC8830+AD7476): 16-bit, <= 20 MHz (AD7476 limits the bus).

1) MCP23017 driver  -  Core/{Inc,Src}/drivers/mcp23017.{h,c}
--------------------------------------------------------------------------------
   WHAT : Bus-agnostic driver for the 16-bit I2C GPIO expander used everywhere
          (2x on Matrix Card, 8x per HV Card).
   HOW  : - Instance binds an I2C handle + 7-bit address (0x20 | A2:A0 strap).
          - Assumes power-on default IOCON.BANK=0 with sequential addressing, so
            a 16-bit access auto-increments port A -> port B.
          - HAL_I2C_Mem_Write/Read, blocking, 100 ms timeout.
          - API: Init (sets HAEN, drives latch low, all pins -> outputs),
            IsReady, WriteReg/ReadReg (8-bit), WriteReg16/ReadReg16,
            SetDirection, SetPullups, WritePins, ReadPins,
            WritePin (single bit via shadow latch = one bus write).
   STATE: Complete. Compiles (HAL-only deps).

2) Matrix Card layer  -  Core/{Inc,Src}/cards/matrix_card.{h,c}
--------------------------------------------------------------------------------
   WHAT : Routes any of 256 harness pins onto the HI or LO analogue bus.
   HOW  : - Addressing confirmed from updated Matrix_Card.pdf:
            * Channel select = SHARED 4-bit bus per bank (HI_S0..3 / LO_S0..3),
              driven by MCU GPIO over the J101 ribbon.
            * Mux select = ONE-HOT enable (HI_EN1..16 from MCP23017 U101=0x20,
              LO_EN1..16 from MCP23017 U102=0x21).
            * Route pin p(1..256): mux=(p-1)>>4, channel=(p-1)&0x0F.
          - MatrixCard_Init (inits both expanders, forces all-open),
            SelectPin(bank,pin) [break-before-make], ConnectPair(hi,lo),
            BankOff, AllOff.
          - Enable polarity #define MATRIX_EN_ACTIVE_LOW (=1). CONFIRMED from
            updated schematic: HI_EN/LO_EN run direct from MCP23017 to CD4067 E
            pin (pin 15, active-low), no buffer/inverter. Init forces all-open
            because generic MCP Init leaves outputs low (= active-low muxes ON).
          - Pin->(mux,channel) is a single linear function matrix_map_pin();
            swap for a LUT if PCB routes I0..I15 differently. NOT yet verifiable
            from the PDF (muxer sheet too dense) - confirm from netlist.
          - Select GPIOs tolerate NULL ports (usable before the Control-Card
            connector is wired).
   STATE: Code complete. Verified against updated Matrix_Card.pdf. One assumption
          left (pin->mux/channel mapping); see OPEN ITEMS.
   NOTE : Updated schematic removed pull resistors on the EN lines (only S0..3 +
          LO_COM are pulled down now) -> mux enables FLOAT at power-up until the
          MCP23017s are initialised. Minor power-up safety note (HW).

3) AD7476 driver  -  Core/{Inc,Src}/drivers/ad7476.{h,c}
--------------------------------------------------------------------------------
   WHAT : SPI read-only driver for the 12-bit SAR ADC. Reusable: Control-Card
          front-end ADC (now on SPI1) AND each HV-Card leakage/Vmon ADC (SPI3).
   HOW  : - Instance binds SPI handle + soft-NSS CS GPIO (port/pin).
          - CS low starts conversion; reads one 16-bit frame, masks low 12 bits.
          - API: Init (parks CS high), ReadRaw, CodeToVolts(vref), ReadVolts.
   STATE: Code complete. Requires SPI1 set to 16-bit / <=20 MHz / soft CS in
          CubeMX (see PER-BUS SPI SETTINGS). CS pin TBD (reuse PC2 or dedicate).

6) Logging + RTOS task architecture (this session)
--------------------------------------------------------------------------------
   Logging  Core/{Inc,Src}/app/log.{h,c}
     - Levels ERROR/WARN/INFO/DEBUG, compile-time gated by LOG_LEVEL.
     - Non-blocking: LOG_x() formats a short "<L>[tick] tag: msg" line and drops
       it in a queue (drop-if-full, ISR-safe) -> never stalls a task. A low-prio
       logger task drains the queue to the UART.
     - UART-agnostic; Log_HwInit_LPUART1() brings up LPUART1 on PA2/PA3 (AF12,
       115200) = the Nucleo ST-LINK VCP, so logs stream over the USB cable.
   Tasks    Core/{Inc,Src}/app/tasks.{h,c}  (cmsis_os2)
     tSafety(High)  - forces safe-all on fault; backstops HV-disable when idle;
                      IWDG-refresh hook (TODO when watchdog enabled).
     tSequencer(Nrm)- runs one test command at a time under the HW mutex.
     tComms(BelowN) - VCP console: c/k/i = continuity/kelvin/insulation,
                      s = force-safe, f = fault, r = clear fault.
     tLogger(Low)   - drains the log queue.
     - Single HW mutex serialises all shared I2C/SPI access (sequencer+safety).
     - Wired via Tasks_Init() in app_freertos.c (USER CODE RTOS_THREADS).
   Free UART decision: USART2/3 + UART4 are in the schematic; free peripherals
   are USART1 / UART5 / LPUART1. LPUART1 chosen (free AND reaches the Nucleo VCP
   on PA2/PA3). In the PRODUCT those pins are USART2 -> production logging UART
   is an open pin-budget item.
